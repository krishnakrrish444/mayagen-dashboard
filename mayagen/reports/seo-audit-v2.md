### 1.2. URL & IA Principles

1. **Single Responsibility URL per Tool**  
   Each tool has:
   - A **clean, short, intent‑rich slug** (e.g., `/logo-maker/`, not `/tools/logo-ai-maker-pro`)
   - Direct crawl path from: `/suite/` → relevant suite hub → tool page.

2. **Category Hubs as SEO Pillars**  
   Category pages target broad, high‑volume keywords and internally link to:
   - Tool pages (Tier 1 and Tier 2)
   - Best‑practice guides (blog posts)
   - Templates filtered for that category

3. **Templates & Examples as Long‑Tail Capture**  
   For each major category:
   - `/templates/logos/`
   - `/templates/instagram-posts/`
   - `/templates/flyers/`
   - etc.

4. **Language‑Aware URLs (if/when localized)**  
   When you localize:
   - Use subfolders: `/es/logo-maker/`, `/de/flyer-maker/`
   - Hreflang between language variants
   - Keep slugs translated where it helps CTR, but maintain canonical to language‑specific page.

5. **Avoid Query‑String‑Only Tools**  
   Every meaningful tool must be accessible via a **crawlable path**, not only behind app states or UTM parameters.

## 3. Technical SEO Priorities (Next.js Migration)

Assume migration to **Next.js (App Router)** with SSR/SSG and API routes. Priorities are tuned to the **suite‑first** view and tool‑based IA.

### 3.1. Crawlability & Indexation

1. **Robots.txt**
   - Allow all by default.
   - Explicitly disallow internal app states (e.g., `/app/`, `/dashboard/`, `/editor/*` if they are non‑indexable UIs).
   - Include sitemap references:
     ```
     User-agent: *
     Disallow: /api/
     Disallow: /dashboard/
     Disallow: /editor/
     Allow: /

     Sitemap: https://mayagen.ai/sitemap.xml
     Sitemap: https://mayagen.ai/sitemap-tools.xml
     Sitemap: https://mayagen.ai/sitemap-blog.xml
     ```

2. **Sitemap Strategy**
   - `sitemap.xml` → index of:
     - `sitemap-static.xml` (home, suite, pricing, about)
     - `sitemap-tools.xml` (all tool pages, category hubs)
     - `sitemap-templates.xml` (templates, if many)
     - `sitemap-blog.xml` (blog)
   - Generate via Next.js route handlers with caching.

3. **Canonical URLs**
   - For every tool and category route, set `<link rel="canonical" href="https://mayagen.ai/..." />`.
   - Avoid creating multiple paths to the same tool (`/tools/logo-maker` vs `/logo-maker`); pick one canonical structure.

4. **Paginated & Filtered Views**
   - For templates and gallery pages, use **canonical to root listing** when filters are purely UX.
   - Use query params (e.g., `?platform=instagram`) but canonical stays on `/templates/social-media/` unless it’s a distinct SEO page.

### 3.2. Rendering & Performance

1. **SSR/SSG for SEO Pages**
   - Home, suite page, all category hubs, and all flagship tool pages should use SSG/ISR with quick TTFB.
   - Tool UIs can hydrate client‑side but core copy and headings must be visible in initial HTML.

2. **Core Web Vitals**
   - Use Next/Image with appropriate `sizes` and lazy loading for galleries.
   - Minimize layout shifts in tool UIs; prefer reserving space with fixed container sizes.
   - Use RUM monitoring (e.g., Vercel Analytics, custom CWV tracking) to catch regressions.

3. **Internationalization (Future‑Proofing)**
   - Implement i18n routing from the start with Next.js `app/[locale]/` pattern.
   - Even if only `en` is active now, this enables later expansion.

### 3.3. Metadata, OG & Twitter Cards

Implement via **Next.js Metadata API** (`generateMetadata` / `metadata` export).

**For each page type**:

1. **Home (`/`)**
   - Title: `Mayagen – AI Creative Suite for Logos, Social Media, Ads & More`
   - Description: mention 45+ tools, 100+ languages, scheduling.
   - OG image: branded image showing multiple content types (logo, meme, post, ad).

2. **Suite Page (`/suite/`)**
   - Title: `Mayagen AI Creative Suite – Design, Write & Schedule Content`
   - Description: emphasize “all‑in‑one social media content" and “AI creative workspace”.

3. **Category Hubs**
   - Titles like: `Social Media Content Suite – AI Posts, Stories & Reels | Mayagen`
   - OG image showing category‑specific UI.

4. **Tool Pages**
   - Use templated but human‑edited metadata:
     - Title: `{Tool Name} – {Primary Keyword} | Mayagen`
     - Description: 140–160 chars summarizing value.
   - OG: show example outputs for that tool.

Ensure Twitter cards use `summary_large_image`.

### 3.4. Structured Data (Schema.org)

1. **Organization Schema (Global)**
   - On all pages via JSON‑LD:
     - `@type`: `Organization`
     - `name`, `url`, `logo`, `sameAs` (social profiles).

2. **Product / SoftwareApplication Schema for the Suite**
   - On `/` and `/suite/`:
     - `@type`: `SoftwareApplication`
     - `applicationCategory`: `DesignApplication` / `ProductivityApplication`
     - `offers`: subscription tiers with price range.

3. **Tool Pages – SoftwareApplication / WebApplication**
   - For each flagship tool:
     - `@type`: `WebApplication`
     - `name`: e.g., `Mayagen Logo Maker`
     - `applicationCategory`: e.g., `GraphicDesignApplication`
     - `operatingSystem`: `Web`
     - `isPartOf`: `https://mayagen.ai/suite/`

4. **HowTo / FAQ Schema**
   - For tools where a “how to” section is clear (logo, flyer, meme, product mockups):
     - Use `HowTo` structured data describing step‑by‑step usage.
   - Use `FAQPage` schema for FAQ sections (2–6 Q&As per page).

5. **Article Schema for Blog**
   - Standard `BlogPosting` for content marketing pieces.

### 3.5. Open Graph for Suite View & App States

- For logged‑in app views (`/app/*`), consider **noindex** and simpler metadata.
- Shareable design links (if any) should:
  - Have OG images generated via an image proxy service (e.g., dynamic OG showing the design).
  - Use `og:type = website` with relevant snippet text.

### 3.6. Other Technical Considerations

1. **Canonical vs. App Routes**
   - Tools should have a marketing page (`/logo-maker/`) and, optionally, an app route (`/app/logo-maker`).
   - Marketing page: indexable, keyword‑focused, strong copy.
   - App route: `noindex`, canonical to marketing page if needed.

2. **Handling Legacy URLs**
   - Implement 301 redirects from old paths to new IA:
     - `/tools/logo` → `/logo-maker/`
     - `/graphics/memes` → `/meme-generator/`
   - Maintain a redirect map in code.

3. **Analytics & Event Tracking**
   - Track from which **tool landing pages** sign‑ups happen.
   - Use this data to prioritize content and CRO on high‑converting tools.
